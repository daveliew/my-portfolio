### Arc 3: Technical Depth (Weeks 6-8)

#### Post #5: "Choosing Your Architecture: Tool Use â†’ RAG â†’ Agents â†’ MCP"
**Word count**: 1500-2000 words

**Content outline**:
1. **Introduction**: The complexity spectrum
2. **Decision Matrix**: Problem complexity vs solution pattern
3. **Level 1: Simple Tool Use**
   - When: Single function, deterministic, low complexity
   - Cost: $0.01-0.10 per query
   - Example: Weather lookup, simple calculations
4. **Level 2: RAG (Retrieval Augmented Generation)**
   - When: Knowledge access, semantic search, contextualized responses
   - Cost: $0.10-0.50 per query
   - Example: Documentation Q&A, support knowledge base
5. **Level 3: Agents (Autonomous Reasoning)**
   - When: Multi-step workflows, decision-making, dynamic tool selection
   - Cost: $0.50-5.00 per session
   - Example: Research tasks, customer lifecycle automation
6. **Level 4: MCP (Model Context Protocol)**
   - When: Scale, standardization, cross-project reusability
   - Cost: Infrastructure investment + per-query costs
   - Example: 15+ tools, multi-agent orchestration
7. **When to Graduate**: Signs you need the next level
8. **Anti-patterns**: Over-engineering (MCP for 1 tool), under-engineering (tool use for complex workflows)
9. **Real Examples**: 3 client scenarios, which pattern chosen, why
10. **My Journey**: "Every architecture mistake I made, what they cost"
11. **Productive Tension**: Simplicity vs power (when complexity pays off)
12. **Key Takeaways**: Decision matrix, cost considerations, graduation signals
